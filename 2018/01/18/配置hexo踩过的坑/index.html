配置hexo踩过的坑

发表于 2018-01-18
字数统计: 401 | 阅读时长 ≈ 2

关于hexo的配置网上有很多教程，这里推荐两个我参考过的教程：

https://zhuanlan.zhihu.com/p/26625249
https://www.jianshu.com/p/189fd945f38f

虽然很详细很全面，但是过程中还是遇到各种坑，这里记录一下踩坑的过程。

1.github仓库名的格式必须为：用户名.guthub.io,比如我的仓库名为： brothertwo.github.io

2.配置站点文件和主题文件的时候，注意大部分#号后有一个空格，冒号后面也有一个空格，hexo的检查非常严格，缺少空格和大小写不符都会报错：

deploy:
  type: git
  repo: git@github.com:BrotherTwo/brothertwo.github.io.git
  branch: master

3.使用npm install -g hexo-cli 安装完hexo后，对hexo的一些列配置操作就需要进入hexo文件夹内进行，否则无效

4.配置完成后，是使用hexo s开启本地服务器，但是访问localhost:4000端口却报404，原因是福昕阅读器占用了4000端口，可以终止福昕阅读器的进程或者用hexo s -p 5000启动服务器，但是每次启动都需要这么做，非常麻烦，所以进入站点配置文件，添加以下代码：

server:
  port: 5000
  compress: true
  header: true

5.主题的语言不在主题配置文件中修改，而是在站点配置文件中。同样作者，描述之类的属性，都是在站点配置文件中修改

6.文章内容超长时候，可以在md文件中添加<!-- more -->作为断点，当生成文章后，会出现阅读全文的图标

hexo
Hello World
